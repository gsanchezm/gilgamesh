> **For agentic workers:** REQUIRED SUB-SKILL: the orchestrator runs **Phase 0** (keystone v0.7) inline in this session, then dispatches **Phase 1** as three parallel worktree subagents (the project's v2‚Äìv8 pattern: `superpowers:using-git-worktrees` + `superpowers:dispatching-parallel-agents`), then runs **Phase 2** (serialized integration) inline. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship three disjoint slices ‚Äî 41 Stripe partial refunds, 42 voice (STT/TTS) in chat, 43 Reports per-tool breakdown ‚Äî behind a keystone v0.7 amendment (`RunResult.tool`/`discipline`) landed in series first; charter the mobile app as a separate program.

**Architecture:** Clean Architecture ports & adapters, per-`orgId` tenant isolation. Phase 0 amends the frozen keystone + migrates `run_results` on `main` in series so Slice 43 builds against a frozen contract and the shared dev DB never races a worktree. Phases 1's three slices touch disjoint domains (billing / chat-voice / reports) ‚Äî the only shared-file collision (`schema.prisma`) is fully absorbed by Phase 0.

**Tech Stack:** TypeScript (ESM domain/app/ui/web, CommonJS Nest api), Prisma 6 / Postgres 16 + pgvector, NestJS, React 19 + Vite, Vitest 3, Cucumber-js (BDD), Playwright (e2e), Stripe SDK, Azure Speech (voice, behind a port; stub in CI).

## Global Constraints

- **Methodology:** every slice is SDD‚ÜíBDD‚ÜíTDD ‚Äî spec in `specs/slices/NN-*` ‚Üí Gherkin `.feature` ‚Üí failing unit tests ‚Üí green ‚Üí refactor. Domain has **zero** framework imports.
- **Tenant isolation:** every query is per-`orgId`; a non-member gets `NOT_FOUND` (never 403). Secrets are vault refs, never raw tokens; provider keys never reach logs/errors/rows.
- **Subagent verification (Tier-0 shared Postgres/Redis/ports): Docker-free only** ‚Äî `pnpm -r typecheck`, `pnpm -r lint` (where present), `pnpm -r test`. Subagents MUST NOT run `test:int` / `test:bdd` / Playwright (shared DB/ports). The orchestrator runs those serialized in Phase 2.
- **Offline harnesses:** all four test harnesses pin `BRAIN_MODE=SSO_MODE=EMAIL_MODE=PAYMENTS_MODE=VAULT_MODE=offline`; Slice 42 adds `VOICE_MODE=offline` to all four (`vitest.config`, `apps/api/acceptance/cucumber.cjs`, `apps/web/playwright.config.ts`, api test-setup).
- **Additive ports only:** Slices 41/42 add methods/ports without a keystone amendment (the S13/S34/S40 precedent). Only Phase 0 amends the keystone (v0.7).
- **Naming:** slice dirs `specs/slices/41-stripe-refunds`, `42-voice`, `43-reports-per-tool`. EmptyState titles are **period-less** (the v6 lesson). Commit trailers per repo convention.
- **Honesty:** the per-tool breakdown renders `DeterministicKernel`-emitted (stub) tool/discipline until the real TOM kernel lands ‚Äî state it in the spec and the data path.
